## Why does this repository qualify? (max 500 chars)

This project helps OSS maintainers and AI builders evaluate tools before adoption by turning noisy social posts, GitHub repositories, and product claims into structured evidence packs. It separates claims, repo metadata, verification status, and hype signals so maintainers can avoid adopting dependencies based on unverified marketing claims.
